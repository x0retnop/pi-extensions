# pi-project-memory

**Проектная память для Pi.** Семантический поиск по сохранённым фактам проекта и ручное управление записями через TUI.

## Суть

Каждый проект получает своё хранилище: решения, паттерны, грабли — в `facts`, открытые задачи — в `todos`. Агенты семантически ищут по `facts`, чтобы вспомнить, «как у нас сделано». Ничего не сохраняется автоматически — только явный вызов команды.

## Как это работает

1. **`.project-id`** в корне проекта (одна строка, например `my-app`). По ней Pi понимает, к какому проекту привязать карточки.
2. **Факты** (`facts`) — вечные: архитектура, решения, паттерны, грабли. Индексируются в sqlite-vec, доступны к семантическому поиску.
3. **Туду** (`todos`) — задачи между сессиями. Не индексируются, только JSONL-список.

## Быстрый старт

```bash
# 1. Включить в бэкенде (0x010/.env)
PROJECT_MEMORY_ENABLED=true

# 2. Скопировать расширение
# C:\10x001\pi extensions\pi-project-memory  →  ~/.pi/agent/extensions/

# 3. В корне проекта
my-app > .project-id
```

Перезапустить Pi и бэкенд.

По умолчанию расширение стучится на `http://127.0.0.1:8000`. Переопределить можно через env:

```bash
export PI_PROJECT_MEMORY_URL="http://127.0.0.1:8000"
# или
export PI_BACKEND_URL="http://127.0.0.1:8000"
```

## Команды

- `/pm` — интерактивный TUI: поиск, просмотр, добавление, редактирование, удаление фактов и todos.
- `/remember type|topic|what` — быстро сохранить факт. Типы: `decision`, `pattern`, `gotcha`, `architecture`, `bugfix`.
- `/todo topic|what` — добавить задачу.
- `/done` — извлечь факты из текущей сессии с помощью локальной LLM, выбрать какие сохранить.

## Инструменты агента

- `project_facts({ query })` — семантический поиск по фактам проекта.
- `project_facts({ recent: true, limit: 20 })` — последние факты для аудита.
- `curate_facts(...)` — включается вручную через `/pm` для редактирования/слияния/удаления фактов.

## Дисциплина сохранения

Сохраняй только то, что пригодится через месяц:

- ✅ решения и архитектура
- ✅ грабли, из-за которых возник баг
- ✅ открытые задачи
- ❌ не писать: опечатки, стиль, очевидное из кода, разовые правки

Ключевой тест: «Поможет ли это будущему агенту через 30 дней?"
