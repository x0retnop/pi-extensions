## Что меняет всё: "Fact Cards" вместо статей

Вместо summary/detail — **атомарные карточки**. Это проще писать, проще читать, проще искать.

```json
{
  "item_id": "pm-abc",
  "project_id": "pi-extensions",
  "type": "decision",
  "topic": "Extension dependencies",
  "what": "Never bundle @earendil-works/*; list as peerDependencies only",
  "why": "Pi provides core packages at runtime; bundling causes version conflicts",
  "where": ["docs/creating-extensions.md", "pi-session-memory/package.json"],
  "tags": ["dependencies", "workflow"],
  "source_session": "...",
  "created_at": "..."
}
```

**Почему это работает:**
- `topic` = 2-4 слова. Понятно с первого взгляда.
- `what` = 1 предложение. Можно вставить в prompt без перегрузки.
- `where` = файлы. Агент сразу знает, куда идти за деталями.
- `type` позволяет фильтровать: `decision`, `pattern`, `architecture`, `gotcha`, `todo`.

**Векторная БД индексирует:** `text = topic + "\n" + what + "\n" + why + "\n" + tags.join(", ")`

## Как заставить агента звать search (триггеры)

PromptGuidelines должны быть **конкретными** и **исключающими**:

```
TRIGGERS — call project_memory_search when:
  - User asks about project conventions, architecture, or "how do we do X here"
  - You are about to read 3+ files just to understand project structure
  - User refers to a past decision or pattern ("as we decided", "keep consistent")

NEVER call for:
  - Specific file content (use read/grep instead)
  - Code review of a single file
  - Questions answered by the current session context

WORKFLOW:
  1. Search → get list of topic+what cards (3-5 hits).
  2. If card is relevant but what is insufficient → get(item_id) for full detail.
  3. After making a non-trivial architectural decision, non-obvious bugfix, or discovering a gotcha → add().
```

**Ключевое:** `search` выдаёт **cards**, не **тексты**. Агент за 1 взгляд решает: релевантно / не релевантно.

## Что может сломаться

1. **Extract quality.** LLM может выдать мусор или пропустить важное. Митигация: промпт должен быть жёстким + fallback на explicit `project_memory_add`.
2. **Privacy/sensitivity.** Весь transcript уходит на 0x010 сервер. Если там конфиденциальные данные — проблема. Митигация: extract делать только если `PROJECT_MEMORY_AUTO_EXTRACT=1` в env.
