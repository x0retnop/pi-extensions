## Key Decisions & Architecture Notes
- **Runner Stability:** Massive stability improvements were implemented in `runner.ts`, including capping `stderr` output at 100 KB (`STDERR_CAP`), ensuring proper cleanup of `SIGKILL`/`AbortSignal` timers and listeners, and using recursive temporary file deletion (`fs.promises.rm`).
- **Chain Mode Context:** The issue where the second agent ignored `{previous}` was resolved by updating agent prompts (`scout-gemma.md`, `index.ts`) to include a strong guideline forcing agents to trust and utilize the previous output context.
- **TUI Functionality:** `tui.ts` was completely rewritten to support `single`, `parallel`, and `chain` execution modes. The history structure (`HistoryEntry`) was updated to include a `steps` array, enabling complex rerun functionality for multi-step tasks.
- **Agent Configuration:** `agents.ts` was updated to support `extensions` frontmatter, allowing agents to be configured to run with specific extension sets.
