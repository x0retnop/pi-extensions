## Key Decisions & Architecture Notes
- **Runner Stability:** Crucial resource management fixes were implemented in `runner.ts`. This included capping `stderr` output to `100 KB`, ensuring proper cleanup of `SIGKILL`/`AbortSignal` timers and listeners, and using `fs.promises.rm` for robust temporary file deletion.
- **TUI Input Improvement:** The interactive settings UI was refactored using helper functions (`editString`, `pickMany`) to replace unreliable `ctx.ui.input` with `ctx.ui.editor` for prefilled values and implemented a custom multi-select loop for handling custom extensions.
- **Agent Robustness:** All subagents utilize detailed, multi-section system prompts (e.g., `prompts/handoff-gemma-prompt.md`) to ensure high-quality, actionable, and context-aware outputs.
